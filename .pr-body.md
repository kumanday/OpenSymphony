## Summary

Compose services for LiteLLM, Postgres, Prometheus, and Grafana with local persistence and healthchecks.

## Deliverables

- ✅ docker-compose.yml
- ✅ .env.example (Example env file)
- ✅ configs/prometheus/prometheus.yml (Prometheus scrape config)
- ✅ configs/grafana/provisioning/ (Grafana provisioning stubs)
- ✅ configs/litellm/config.yaml

## Acceptance Criteria

- ✅ One command starts the local stack (`docker compose up -d`)
- ✅ LiteLLM is healthy (responds to health/liveliness endpoint)
- ✅ Prometheus scrapes LiteLLM metrics (verified via /api/v1/targets API)
- ✅ Grafana starts with a working datasource (Prometheus datasource provisioned automatically)

## Services

| Service | Port | Health Check |
|---------|------|--------------|
| Postgres | 5432 | pg_isready |
| LiteLLM | 4000 | HTTP health endpoint |
| Prometheus | 9090 | HTTP /-/healthy |
| Grafana | 3000 | HTTP /api/health |

## Quick Start

1. Copy `.env.example` to `.env` and customize if needed:
   ```bash
   cp .env.example .env
   ```

2. Start the stack:
   ```bash
   docker compose up -d
   ```

3. Access the services:
   - LiteLLM UI: http://localhost:4000/ui
   - Prometheus: http://localhost:9090
   - Grafana: http://localhost:3000 (admin/admin)

4. Stop the stack:
   ```bash
   docker compose down
   ```
